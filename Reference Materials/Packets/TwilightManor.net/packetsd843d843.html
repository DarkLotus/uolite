Sent By: Client0x34 - Character Skills/Stats Request (10 bytes)
Variable TypeStart IndexPacket/Variable Name
byte 0 Packet ID (0x34 - Character Skills/Stats Request)
int 1 Pattern (0xedededed)
byte 5 Request type (4 for stats/status bar, 5 for skills)
uint 6 UID the client is requesting information about
The only valid request types I've seen are 4 and 5. The client appears to send 4 whether it wants the HP bar, party stats bar, or full status window - It's up to the server to send what it wants. The client may send this packet twice in one TCP packet when requesting the status bar for its character.

If the request type is 4, reply with packet 0x11. If the request type is 5, reply with packet 0x3a.
