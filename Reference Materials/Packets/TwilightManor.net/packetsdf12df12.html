Sent By: Client0x80 - Login & Request Shard List (62 bytes)
Variable TypeStart IndexPacket/Variable Name
byte 0 Packet ID (0x80 - Login & Request Shard List)
char[30] 1 Account Name
char[30] 31 Account Password
When the client connects to the server, it always sends four bytes first, which are the client's local IP, but are intended to be used as the encryption seed. They're ignorable unless you're using encrypted clients.

Upon the first connection attempt to a server, the client sends this packet, which comes immediately after the four initialization bytes, though whether it is in its own TCP packet or not depends on the client sending. It is safe to disgregard the account name and pass in the packet and just send the server list, unless you don't want people without accounts to even see the name of your shard.

Server should respond with packet 0xA8
